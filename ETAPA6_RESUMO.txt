📋 OBJETIVO
───────────
Otimizar o carregamento de fontes TTF, evitando leituras repetidas do disco
a cada geração de certificado.

✅ ANÁLISE REALIZADA
────────────────────
1. ✓ Analisado arquivo: src/services/fontService.js
2. ✓ Identificado problema: fs.readFileSync() chamado a cada requisição
3. ✓ Verificada existência de outras operações de I/O pesadas
4. ✓ Nenhuma outra operação de I/O encontrada no código-fonte

🔧 OTIMIZAÇÕES IMPLEMENTADAS
──────────────────────────────

1. CACHE DE BYTES DE FONTES (NOVO)
   ├─ Implementado IIFE (Immediately Invoked Function Expression)
   ├─ Executa fs.readFileSync() UMA ÚNICA VEZ na inicialização do módulo
   ├─ Armazena bytes em memória no objeto FONT_BYTES_CACHE
   └─ Todas as 6 fontes pré-carregadas no startup

2. REUTILIZAÇÃO DE BYTES
   ├─ Função carregarFontes() agora usa FONT_BYTES_CACHE
   ├─ Elimina I/O de disco repetido a cada requisição
   └─ Mantém fallback para StandardFonts quando arquivo não existe

📊 IMPACTO DE PERFORMANCE
──────────────────────────
• ANTES: 6 × fs.readFileSync() por requisição = I/O lento
• DEPOIS: 0 × fs.readFileSync() por requisição = I/O rápido

Estimativa:
  - Cada leitura de fonte: ~100-650 KB (6 fontes = ~2.4 MB)
  - Requisições simultâneas: sem contenção (bytes em memória)
  - Tempo economizado: 10-50ms por requisição (dependendo I/O)
